About This Project

This project aims to detect network anomalies using machine learning techniques.

Objective

The goal of this project is to identify and alert users about any abnormal network behavior, such as Distributed Denial of Service (DDoS) attacks, port scanning, and unauthorized data exfiltration.

Technology Stack
  • Python
  • Flask
  • TensorFlow
  • Bootstrap
  • Machine Learning
Key Features
  • Real-time network packet capture
  • Anomaly detection using a trained machine learning model
  • Flask web interface for easy interaction
  • Scalable and easily extendable for additional features

How Scanning and Anomaly Detection Work

This section explains how the network scanning and anomaly detection process works in real-time, ensuring a robust security mechanism.

Step 1: Network Packet Capture

The process starts by capturing live network packets using **Wireshark** or **tcpdump**, depending on your system configuration. The packets include various network behaviors such as source/destination IP, packet size, protocol, and more.

Step 2: Preprocessing

Captured packets are preprocessed by converting the raw data into structured information. This includes converting timestamps into seconds and one-hot encoding categorical values like protocols, ensuring the data is ready for the model.

Step 3: Feature Scaling

The feature values are then normalized using **StandardScaler** to ensure all features have equal influence on the model. This helps the model learn more effectively and improves performance.

Step 4: Model Prediction

The preprocessed data is fed into a **trained machine learning model**, which uses deep learning algorithms (such as a neural network). The model predicts whether the observed network traffic is normal or anomalous based on patterns learned during training.

Step 5: Anomaly Detection

If the model detects an anomaly, it flags the network traffic and outputs the anomaly count. The detected anomalies could include suspicious behaviors such as a potential DDoS attack or unusual data transfer patterns.

Why Is This Important?

Network anomalies can often indicate a security breach or attack, which can lead to data loss or downtime. Detecting these anomalies in real-time ensures timely response and protects sensitive information.
